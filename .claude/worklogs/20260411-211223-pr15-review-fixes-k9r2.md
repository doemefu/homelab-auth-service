```sql
WHERE (
    (refresh_token_expires_at IS NOT NULL OR ... OR device_code_expires_at IS NOT NULL)
    AND GREATEST(COALESCE(...)) < NOW()
)
OR (all-NULL AND authorization_code_issued_at < NOW() - INTERVAL '24 hours')
```

### Change 5 — AbstractIntegrationTest.java

Replaced static initializer with `@Testcontainers` + `@Container`:
```java
@Testcontainers
public abstract class AbstractIntegrationTest {
    @Container
    static final PostgreSQLContainer<?> postgres = new PostgreSQLContainer<>("postgres:17-alpine");
```

**Important context:** Commit `0b26860` previously added the static initializer as a "fix". However, the real problem with static initializer is that multiple Spring contexts (loaded for different test classes with different `@DynamicPropertySource` properties) cause the container to become unavailable for later test classes. The `@Testcontainers` + `@Container` approach gives proper JUnit 5 lifecycle management.

**CI failure explanation:** The `ffed4a2` commit (CSRF fix, before our changes) was already failing CI with `OidcFlowIntegrationTest` and `FlywayMigrationTest` unable to connect to Postgres. This is because the static initializer started Postgres OK for the first Spring context (AuthServiceApplicationTests) but the container became unresponsive by the time the second context (OidcFlowIntegrationTest with extra OIDC properties) loaded ~5 minutes later. Our `@Testcontainers` fix addresses this.

### Key lessons learned

1. **Never use static initializer for Testcontainers in abstract base classes** — `@Testcontainers` + `@Container` is the correct approach. Static initializer leaks resources and breaks when multiple Spring contexts load in sequence.
2. **COALESCE(col, to_timestamp(0)) inside GREATEST** treats all-NULL rows as expired (epoch < NOW). Always guard with IS NOT NULL when mixing COALESCE-with-default and conditional logic.
3. **{noop} prefix is mandatory for OIDC client secrets** with DelegatingPasswordEncoder — without it the secret is treated as BCrypt and auth fails silently at /oauth2/token.
4. **Jackson 3 uses `tools.jackson` group** (not `com.fasterxml.jackson`) — always verify imports when subagents add new Jackson usage.
