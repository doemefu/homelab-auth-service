### SB4 / SS7 lessons learned (key fixes applied in this session)

| Fix | Root cause |
|-----|-----------|
| Mock `JwtService` not `JwtAuthenticationFilter` | `OncePerRequestFilter.doFilter()` is `final` — Mockito can't intercept it; mocked `doFilterInternal` does nothing, breaking the chain |
| `@Import(SecurityConfig.class)` in `UserControllerTest` | `@WebMvcTest` component-scans only web-layer types; `@Configuration` beans are excluded unless explicitly imported |
| Replace `@WithMockUser` with `SecurityMockMvcRequestPostProcessors.user(...)` | Spring Security 7's `SecurityContextHolderFilter` replaces `SecurityContextPersistenceFilter`; with `STATELESS` policy it loads an empty context from `NullSecurityContextRepository` at request start, overwriting the test context |
| `authentication(new UsernamePasswordAuthenticationToken("username", ...))` for principal tests | `@WithMockUser` sets a `UserDetails` principal; `JwtAuthenticationFilter` sets a plain `String` principal — type mismatch causes `@AuthenticationPrincipal String` to resolve null |
