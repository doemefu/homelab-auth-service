## 2. plan

**Goal:** All 16 review findings are resolved, all unit tests pass, code is ready for commit.

### Context

**Summary:** The M1 critical review found 16 issues ranging from critical (`:latest` tag, spec drift) to low (pom metadata). Most are 1-5 line fixes. The two most invasive changes are: (1) refresh token hashing (changes AuthService + migration), and (2) LocalDateTime → Instant migration (touches entities, services, DTOs, tests, Flyway).

**Assumptions:**

| ID | Assumption | Confidence | Implication if wrong |
|----|------------|------------|----------------------|
| A1 | SHA-256 hash of UUID refresh token is sufficient | high | Would need HMAC or bcrypt — but SHA-256 is standard for opaque tokens |
| A2 | `TIMESTAMPTZ` is available in PostgreSQL 17 | high | It is — core type since forever |
| A3 | jjwt 0.12.6 supports `kid` in JWK builder | high | Verified in jjwt docs — `Jwks.builder().key(k).id("kid").build()` |

**Open questions:**

| ID | Question | Severity |
|----|----------|----------|
| Q1 | Update project spec for Java 25/SB 4.0.5? | non-blocker — flag to user in ship notes |

### Options considered

### Changes

**C1 - K8s deployment: remove `latest`, fix `imagePullPolicy`** (findings #1, #16)
Files: `k8s/deployment.yaml`
- Change `image: ghcr.io/doemefu/homelab-auth-service:latest` → `image: ghcr.io/doemefu/homelab-auth-service:SHA_PLACEHOLDER`
- Change `imagePullPolicy: IfNotPresent` → `imagePullPolicy: Always`
- Add comment explaining that SHA is set during deployment

**C2 - GlobalExceptionHandler: handle AccessDeniedException** (finding #4)
Files: `exception/GlobalExceptionHandler.java`
- Add `@ExceptionHandler(AccessDeniedException.class)` returning 403
- Add `@ExceptionHandler(BadCredentialsException.class)` returning 401
- Ensure these are ordered before the generic `Exception` handler

**C3 - JWKS endpoint: add `kid`** (finding #5)
Files: `service/AuthService.java`, `security/JwtService.java`
- Add a static `KID` constant (SHA-256 thumbprint of the public key, or a simpler static ID)
- Set `kid` in JWK: `Jwks.builder().key(publicKey).id(kid).build()`
- Set `kid` in JWT header: `.header().keyId(kid).and()` in token generation
- This enables key rotation for future use

**C4 - Hash refresh tokens before DB storage** (finding #6)
Files: `service/AuthService.java`, `repository/RefreshTokenRepository.java`
- Add private `hashToken(String raw)` method using SHA-256
- On login/refresh: store `hashToken(rawToken)` in DB, return raw token to client
- On refresh lookup: `findByToken(hashToken(rawRefreshToken))`
- No migration needed — existing tokens (none in prod yet) are irrelevant

**C5 - Scheduled cleanup of expired refresh tokens** (finding #7)
Files: `service/AuthService.java`, `repository/RefreshTokenRepository.java`
- Add `@Scheduled(fixedRate = 3600000)` method to delete expired tokens
- Add `@EnableScheduling` to main application class
- Add `deleteByExpiresAtBefore(Instant cutoff)` to repository
- Note: using `@EnableScheduling` is a Spring Boot built-in, not a new dependency

**C6 - LocalDateTime → Instant + TIMESTAMPTZ** (finding #8)
Files: `entity/User.java`, `entity/RefreshToken.java`, `service/AuthService.java`, `dto/UserResponse.java`, `db/migration/V2__timestamps_to_timestamptz.sql`, all tests
- Change all `LocalDateTime` fields to `Instant`
- Change `LocalDateTime.now()` → `Instant.now()` in entities and services
- Change Flyway V1 `TIMESTAMP` → keep V1 as-is, add V2 migration: `ALTER TABLE users ALTER COLUMN created_at TYPE TIMESTAMPTZ; ...`
- Update `UserResponse` record to use `Instant`
- Update test assertions accordingly

**C7 - Invalidate tokens on username change** (finding #9)
Files: `service/UserService.java`, `repository/RefreshTokenRepository.java`
- In `updateUser()`, when username changes, also delete all refresh tokens for that user
- Inject `RefreshTokenRepository` into `UserService`

**C8 - Service-layer role validation** (finding #11)
Files: `service/UserService.java`
- Add role validation in `createUser()`: if role is not null and not "USER"/"ADMIN", throw `IllegalArgumentException`
- Same in `updateUser()` for role field

**C9 - Remove dead code in logout** (finding #12)
Files: `controller/AuthController.java`
- Remove the `if (username == null)` guard — Spring Security's `anyRequest().authenticated()` ensures only authenticated requests reach this endpoint, so `@AuthenticationPrincipal String username` will never be null

**C10 - Remove `@Component` from `RsaKeyProperties`** (finding #13)
Files: `config/RsaKeyProperties.java`
- Remove `@Component` annotation — `@ConfigurationPropertiesScan` on the main class handles registration

**C11 - Fill pom.xml metadata** (finding #14)
Files: `pom.xml`
- Fill `<name>`, `<description>`, remove empty `<url>`, `<license>`, `<developer>`, `<scm>` blocks

**C12 - Widen password_hash column** (finding #15)
Files: `db/migration/V2__timestamps_to_timestamptz.sql` (combine with C6)
- Add `ALTER TABLE users ALTER COLUMN password_hash TYPE VARCHAR(100);`

**C13 - Update project spec for Java 25/SB 4.0.5** (finding #2)
Files: `../../docs/050-iot-app-rewrite.md`
- Update "Fixed Decisions" table: Java 21 → Java 25
- Update "Tech Stack" table: Spring Boot 3.4.x → 4.0.5
- Update "Pinned Versions" table
- Add note about SB4/SS7 migration

**C14 - Validate integration tests compile** (finding #3)
- Run `./mvnw test-compile` to confirm integration test code compiles
- Run unit tests to confirm no regressions
